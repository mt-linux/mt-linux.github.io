    Advanced Shell Scripting

    Once you're comfortable with basic shell scripts, it's time to go further. Advanced scripting lets you build more powerful and flexible automation tools.

    This page covers:

    • Functions
    • Arguments
    • Logging
    • Debugging
    • Exit codes
    • Trap and cleanup

    ðŸ§  1. Functions

    Functions help you organize and reuse code.

    #!/bin/bash

greet() {
  echo "Hello, $1!"
}

greet "Bishow"

    You can also return values using echo:

    add() {
  result=$(( $1 + $2 ))
  echo $result
}

sum=$(add 3 5)
echo "Sum is $sum"

    ðŸ“¦ 2. Script Arguments

    Shell scripts can accept command-line arguments:

    #!/bin/bash

echo "Script name: $0"
echo "First argument: $1"
echo "Second argument: $2"

    Special Variables

    Variable Meaning
    $0 Script name
    $1, $2, ... Arguments
    $@ All arguments
    $# Number of arguments

    Example:

    #!/bin/bash

if [ $# -lt 2 ]; then
  echo "Usage: $0 <source> <destination>"
  exit 1
fi

cp "$1" "$2"

    ðŸ“‚ 3. Logging Output

    Redirect output to a file:

    #!/bin/bash

log_file="/var/log/myscript.log"

echo "Starting backup..." >> "$log_file" 2>&1
rsync -av ~/Documents /mnt/backup >> "$log_file" 2>&1

    Or use tee to write to both the terminal and the log file:

    echo "Running task..." | tee -a "$log_file"

    ðŸ§ª 4. Exit Codes and Error Handling

    Always check if a command succeeded:

    #!/bin/bash

mkdir /some/dir
if [ $? -ne 0 ]; then
  echo "Failed to create directory!"
  exit 1
fi

    A better way using set -e:

    #!/bin/bash
set -e  # Exit script on any error

echo "Starting..."
cp file.txt /tmp/
echo "Done."

    ðŸ›  5. Debugging Scripts

    Use these techniques to debug:

    • set -x â€” Show each command before running
    • set -e â€” Exit on any error
    • set -u â€” Exit on undefined variables
    #!/bin/bash
set -eux  # enable debugging, exit on error, and undefined vars

echo "Starting script..."
some_command

    You can also run a script with bash -x:

    bash -x myscript.sh

    ðŸ§¹ 6. Cleanup with trap

    trap lets you catch signals like CTRL+C or script exit and run cleanup code.

    #!/bin/bash

cleanup() {
  echo "Cleaning up..."
  rm -f /tmp/tempfile
}

trap cleanup EXIT

touch /tmp/tempfile
echo "Doing work..."
sleep 5

    This ensures /tmp/tempfile is deleted when the script exits.

    ðŸ“š Example: Backup Script with Logging & Error Handling

    #!/bin/bash
set -euo pipefail

LOG="$HOME/backup.log"

backup() {
  echo "[INFO] Starting backup: $(date)" | tee -a "$LOG"
  rsync -av --delete "$HOME/Documents/" "/mnt/backup/Documents/" | tee -a "$LOG"
  echo "[INFO] Backup complete: $(date)" | tee -a "$LOG"
}

trap 'echo "[ERROR] Backup failed at $(date)" | tee -a "$LOG"' ERR

backup
