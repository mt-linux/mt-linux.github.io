    Automating Tasks with Cron Jobs

    Once you’ve written a useful shell script, the next step is to automate it — so it runs on a schedule without your input. That’s where cron jobs come in.

    🧠 What is cron?

    • cron is a built-in Linux utility used to schedule and automate tasks.
    • A cron job is a command or script scheduled to run at a specific time, date, or interval.

    📅 Basic Cron Format

    Each line in a crontab follows this format:

    📅 Cron Format Diagram
    ┌───────────── minute (0 - 59)
│ ┌───────────── hour (0 - 23)
│ │ ┌───────────── day of month (1 - 31)
│ │ │ ┌───────────── month (1 - 12)
│ │ │ │ ┌───────────── day of week (0 - 6) (Sunday=0 or 7)
│ │ │ │ │
* * * * * command-to-run

    Example:

    0 7 * * 1 /home/bishow/scripts/weekly_report.sh

    Runs the script every Monday at 7:00 AM

    ✍️ Editing Your Crontab

    To edit your user’s cron jobs:

    crontab -e

    You'll see or create a list of scheduled commands.

    🧪 Example Cron Jobs

    Schedule Cron Line Meaning
    Every minute * * * * * /home/user/myscript.sh Run the script every minute
    Every day at 7 AM 0 7 * * * /home/user/backup.sh Run daily at 7 AM
    Every Monday 0 9 * * 1 /home/user/weekly_report.sh Run every Monday at 9 AM
    On reboot @reboot /home/user/startup.sh Run once at boot time

    🧹 Best Practices

    • Use absolute paths to scripts and commands:
    /usr/bin/python3 /home/user/script.py
    • Redirect output to log files:
    * * * * * /home/user/cleanup.sh >> /home/user/logs/cleanup.log 2>&1
    • Always test your script manually before scheduling.

    🔍 Viewing and Removing Cron Jobs

    • View your current cron jobs:
    crontab -l
    • Remove your current crontab:
    crontab -r

    🛠 Troubleshooting Tips

    Problem Solution
    Script doesn't run Make sure it's executable (chmod +x)
    Missing environment variables Cron runs in a limited environment — set PATH in the script
    No output Redirect stdout and stderr to a log file

    ✅ Example with Logging

    0 * * * * /home/bishow/scripts/backup.sh >> /home/bishow/logs/backup.log 2>&1

    This runs backup.sh every hour and logs all output.

    📚 More cron Tips

    • Use crontab -e for the current user
    • Use sudo crontab -e for root-level jobs
    • Use systemctl timers on systemd-based systems (more modern alternative to cron)

    🧠 Summary

    • cron helps automate any shell script or command.
    • Great for backups, log rotation, cleanup, reporting, etc.
    • Simple format, powerful results.

    Try scheduling one of your scripts now to run daily — automation is addictive!
